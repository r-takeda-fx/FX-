## 1. 目的（Why）

株式投資に興味はあるが「情報が多すぎて何を見ればいいかわからない」という初心者の課題を解決する。複雑な金融情報を**シンプルに整理**し、投資判断の第一歩を支援する。

## 2. ゴール（What）

### MVP（最小構成）
- 市場サマリーダッシュボード（日経平均・米国市場・先物を一画面で）
- 重要ニュースの自動分類・表示（カテゴリ別）
- 今日の市場の「ひとこと解説」（AIによる要約）

### Nice to Have（後回し）
- おすすめ銘柄機能（Phase 5で実装予定）
- ポートフォリオ管理
- プッシュ通知

## 3. 心理ゴール（Who）

| Before | After |
|--------|-------|
| 「株価とかニュース多すぎて何見ればいいの？」 | 「今日の市場状況、5分で把握できた」 |
| 「FRBとか言われても意味わからない」 | 「このニュースが株価に影響するのね」 |
| 「どの銘柄買えばいいか全然わからない」 | 「初心者向けの銘柄候補がわかった」 |

## 4. 想定障害と対策

| 障害 | 対策 |
|------|------|
| リアルタイム株価APIは高額 | 無料API（Yahoo Finance等）で15分遅延データを使用。初心者には十分 |
| ニュース収集が複雑 | RSS + NewsAPI等を活用。カテゴリは手動定義で開始 |
| おすすめ銘柄の責任問題 | 「投資は自己責任」の免責表示 + 教育目的として提示 |
| 技術スタック選定 | Python + Streamlit で高速MVP開発 |

## 5. アクションプラン

| Phase | 内容 | 成果物 |
|-------|------|--------|
| **1. 基盤** | プロジェクト構築、API調査・選定 | 動作する骨組み |
| **2. 市場データ** | 日経・米国・先物の取得・表示 | ダッシュボード画面 |
| **3. ニュース** | カテゴリ別ニュース取得・表示 | ニュースセクション |
| **4. AI要約** | 市場状況のひとこと解説生成 | 要約機能 |
| **5. 銘柄推薦** | 初心者向け銘柄の提案機能 | おすすめ銘柄セクション |

## 技術構成

```
フロントエンド: Streamlit（Python製、高速プロトタイプ向け）
バックエンド: Python
データ取得: yfinance（無料）、NewsAPI、RSS
AI要約: Claude API または OpenAI API
```
